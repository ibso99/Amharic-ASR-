General Flow

1. Read user settings from train_config.json or command-line arguments.

2. Validate the configuration, paths, durations, batch sizes, and required CSV columns.

3. Load local Amharic audio-text manifests for train, validation, and optionally test.

4. Validate audio paths and remove bad rows, too-short clips, too-long clips, and empty or invalid transcripts.

5. Normalize transcripts so training and evaluation use a consistent Amharic text standard.

6. Convert validated CSV rows into Hugging Face DatasetDict objects.

7. Load WhisperProcessor, which contains both the Whisper feature extractor and tokenizer.

8. Filter transcript labels that exceed Whisper’s decoder limit, usually using max_label_length=440.

9. Keep datasets in lazy mode, meaning log-Mel features are not precomputed for the full dataset.

10. Load WhisperForConditionalGeneration from openai/whisper-small or from a previous checkpoint.

11. Configure Whisper for Amharic transcription using forced decoder prompt IDs.

12. Optionally freeze the encoder or front-end. For the current full fine-tuning workflow, these are not frozen.

13. Enable gradient checkpointing to reduce GPU memory use.

14. Build Seq2SeqTrainingArguments, including batch size, gradient accumulation, evaluation schedule, saving schedule, TensorBoard reporting, and generation settings.

15. Create the custom data collator.

16. During each batch, the collator loads audio from disk, resamples it to 16 kHz, converts it to Whisper log-Mel input features, tokenizes the transcript, pads audio and labels, and masks label padding with -100.

17. Fine-tune Whisper using Seq2SeqTrainer.

18. During evaluation, generate predicted transcripts using model.generate(...).

19. Decode predictions and references back into text.

20. Normalize both predictions and references before metric computation.

21. Compute WER, CER, char_accuracy, and loss.

22. Save checkpoints during training.

23. Load the best checkpoint based on lowest WER.

24. Save the final model and processor to final_model.

25. Optionally run prediction on the test split.
